基于当前框架，在apps目录下再新增一个游戏

---

一、游戏核心循环

凡人 → 修炼 → 突破境界 → 探索机缘 → 获取资源 → 强化自身 → 应对劫难 → 飞升

· 主要行为：修炼（挂机/主动）、探索（随机事件/战斗）、坊市交易、炼丹/炼器、突破瓶颈。
· 长期目标：从凡人修炼到渡劫飞升（多个大境界，每个大境界若干小层）。
· 核心资源：修为（经验值）、灵力（MP，用于技能/炼丹/探索）、气血（HP，战斗与探索消耗）、灵石（货币）、丹药/材料/法宝（道具）。

---

二、世界观与境界体系

还原《凡人修仙传》的经典设定，境界从低到高：

大境界 小层数 突破条件与特点
凡人 1层 初始状态
炼气期 1-13层 修为满即可突破，失败率低
筑基期 初/中/后/圆满 需筑基丹辅助，失败会降修为
结丹期 初/中/后/圆满 需结丹秘法，心魔考验
元婴期 初/中/后/圆满 需元婴丹，渡小天劫
化神期 初/中/后/圆满 需化神丹，雷劫
炼虚期 初/中/后/圆满 空间感悟，大天劫
合体期 初/中/后 极难突破
大乘期 初/中/后 飞升准备
飞升 - 触发飞升结局

突破时随机出现：心魔入侵、天劫降临、灵气反噬等事件，考验玩家资源与运气。

三、主要系统设计

1. 修炼系统

· 被动修炼：实时计时，每秒自动获得微量修为（与灵根资质挂钩）。
· 主动修炼：消耗灵力，获得较多修为，随机触发“顿悟”（倍数暴击）。
· 灵根资质：开局随机生成（伪灵根/三灵根/双灵根/天灵根/异灵根），影响修炼速度与突破成功率，可后天通过奇遇改变。
· 瓶颈机制：每层修为满后需手动“突破”，消耗灵石/丹药，并掷骰判定（随机+资质加成）。

2. 探索系统（核心随机性载体）

· 随机地图：每次进入生成不同场景（山脉、洞府、遗迹、坊市野外等）。
· 事件池：根据境界与场景权重随机抽取事件，例如：
  · 发现药草/矿石（获得炼丹/炼器材料）
  · 遭遇妖兽（进入战斗）
  · 修士争斗（选择观战/参与/逃跑，不同结局）
  · 仙人洞府（概率获得法宝/功法残卷）
  · 空间裂缝（奇遇，高奖励高风险）
  · 心魔幻境（神识判定）
· 探索消耗：每次探索消耗灵力与少量气血，深度探索可遇更高风险与回报。
· 随机战斗：遇敌种类、属性随机，战斗采用回合制文字描述，玩家可选攻击/防御/使用法宝/逃跑，伤害与命中基于自身攻击、敌方防御及随机浮动。

3. 坊市系统

· 随机刷新商品：每次进入坊市或每隔一段时间，商店刷新丹药、材料、法宝、功法残卷等，价格有一定浮动。
· 灵石获取：出售多余材料、击杀妖兽掉落、探索宝箱。
· 特殊商人：极低概率出现神秘修士，出售稀有宝物，可能需要特殊物品交换。
· 物品价格波动：受随机事件影响（如“妖兽潮”导致药材涨价）。

4. 背包与物品系统

· 分类：丹药、材料、法宝、功法、杂物。
· 使用效果：丹药恢复HP/MP、临时提升属性、增加突破概率；法宝可装备，提升攻防或附带技能；功法学习后获得被动效果。
· 容量限制：初始容量小，可通过炼器扩充储物袋。
· 堆叠与耐久：消耗品可堆叠，法宝有耐久度，修理需材料。

5. 炼丹与炼器系统

· 丹方获取：探索、坊市、突破奖励随机获得，丹方也有品阶，不同丹方需求不同材料。
· 炼丹过程：投入材料，根据神识与丹方难度掷骰决定成丹率，可能失败、产出低阶丹药或极品丹药（随机）。
· 炼器：类似，需要矿石与图纸，法宝随机附带属性词条（如“锋利+3”“吸血+2%”）。
· 随机性：即使相同材料，产出结果也有品质波动，鼓励反复尝试。

6. 战斗系统（文字回合制）

· 遇敌触发：探索或特殊事件。
· 战斗界面：弹出层，显示敌人名称、血量、简短描述。
· 动作选项：攻击、防御（减伤）、使用技能/法宝、逃跑（概率成功）。
· 伤害计算：我方攻击×(0.8~1.2随机) - 敌方防御，并考虑属性克制（金木水火土，可选加）。
· 战胜奖励：灵石、材料、概率掉落稀有物品。
· 战败惩罚：损失部分灵石/修为，强制返回安全区，气血保留1点。

7. 天劫与心魔系统（突破瓶颈时触发）

· 天劫：突破大境界时必然触发，多轮雷电/天火/罡风，每轮需要玩家选择应对方式（硬抗/使用法宝/丹药），伤害随机，有概率直接失败。
· 心魔：神识判定，可选择“坚守本心”或“使用清心丹”，失败则突破受阻并临时降低属性。
· 渡劫成功：大幅提升属性，获得稀有奖励。

8. 成就与称号系统

· 隐藏成就：如“首次炼丹失败”“空血反杀”“十连突破无失败”等。
· 称号效果：部分称号提供微弱属性加成，增加收集乐趣。
· 记录展示：在状态栏可查看已获成就。

四、随机性设计精髓

为保证“足够的随机性”，采用以下机制：

1. 加权随机池：事件、掉落、商店刷新均基于多张权重表，且权重随境界/选择动态变化。
2. 伪随机与保底：避免玩家极度脸黑，例如连续10次探索无稀有事件则下次概率提升。
3. 多维度随机：物品属性浮动（±10%）、战斗伤害波动、突破概率波动、炼丹品质波动、商人刷新时间波动。
4. 随机开局：灵根、初始天赋、少量初始资源都有随机范围，增加重玩性。
5. 混沌事件：极小概率触发世界级随机事件（如“灵潮爆发”全服修炼速度翻倍，“魔劫入侵”坊市物价紊乱），单机可通过时间戳触发。

五、手机端适配与交互设计

· 布局：竖屏单列设计，所有内容以卡片形式堆叠，适合拇指操作。
· 底部Tab导航：5个主要功能入口（修炼/探索/坊市/背包/炼丹），固定底部，防止误触。
· 顶部状态栏：始终显示境界、气血、灵力、修为进度条、灵石数量。
· 弹窗交互：战斗、突破、事件选择使用全屏弹窗或底部抽屉，避免页面跳转。
· 长按反馈：部分按钮长按可查看详细信息（如丹药效果预览）。
· 消息日志：可滚动查看最近20条事件记录，关键战斗/突破用醒目颜色。
· 离线收益：利用 localStorage 记录退出时间，再次打开计算离线修炼收益（上限8小时），弹出收获提示。
· 安全区域：适配 iPhone 刘海/底部横条，使用 safe-area-inset-*。

六、技术实现思路（非代码）

· 纯前端：HTML + CSS + JS，单文件或多文件均可，不依赖框架以减小体积。
· 数据存储：使用浏览器 localStorage 保存存档，支持手动导出/导入存档字符串。
· 随机种子：可结合当前时间戳与玩家操作次数生成随机，保证单次游戏内体验。
· 资源图标：使用 Emoji 或 CSS 绘制简易图标，避免外链图片。
· 动画：使用 CSS 过渡/动画处理战斗抖动、数值变化闪烁等，增强反馈。
· 部署：将文件放在 GitHub 仓库的 docs/ 或根目录，开启 Pages，即可通过手机浏览器访问。

八、设计原则总结

1. 碎片时间友好：单次操作几秒，修炼可挂机。
2. 正反馈强烈：突破、爆宝、奇遇必有醒目动画与文字。
3. 风险与机遇并存：探索可能遇险损失，也可能一夜暴富。
4. 足够随机，但不致死：重要损失给予挽回途径（如复活丹、重修buff）。
5. 修仙味要正：文案模仿原作语境，境界名称、事件描述有仙侠气质。
