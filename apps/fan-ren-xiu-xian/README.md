# 凡人修仙录 · Mortal Cultivation

一款致敬《凡人修仙传》的**文字修仙挂机**网页游戏。从一介凡人起步，修炼、突破境界、探索机缘、炼丹炼器、渡劫飞升，体验一段完整的修仙旅程。

技术栈：**纯原生 HTML + CSS + JavaScript（无框架）**，移动端竖屏单列设计，Canvas 都不用——一切皆 DOM 卡片。体积小（构建后 JS ≈ 60KB），加载快。

## 本地运行

```bash
npm install
npm run dev      # 开发服务器 http://localhost:5174
npm run build    # 生产构建到 dist/
npm run test     # 纯逻辑自测（126+ 断言，不依赖浏览器）
```

也可由主框架（落地页）以 `createGame(parent)` 动态挂载，无需独立部署。

## 核心玩法

- **创角**：进入游戏先选男女，随机生成**灵根 / 天赋（1~3 个）/ 气运 / 出身**与**形象**（10+ 预设头像），不满意可反复重随，称心后取名入道。天赋影响修炼、突破、炼丹、攻防与每日活力；气运影响奇遇保底与稀有概率。
- **人物档案**：点状态栏头像随时查看形象、灵根、气运、天赋、出身与各项属性。
- **境界体系**：凡人 → 炼气（13 层）→ 筑基 → 结丹 → 元婴 → 化神 → 炼虚 → 合体 → 大乘 → **飞升**，完整还原原作层级。
- **修炼**：被动实时挂机攒修为；主动修炼消耗灵力与活力、有概率「顿悟」三倍暴击。开局随机灵根（伪/三/双/天/异），影响修炼速度与突破。
- **突破**：小境界掷骰判定（可服突破丹）；大境界需**渡心魔**（神识多轮）或**渡天劫**（多轮雷火，可硬抗 / 祭法宝 / 服丹）。
- **探索**：随机场景（灵山 / 洞府 / 遗迹 / 野外）+ 加权事件池——采药挖矿、遭遇妖兽、修士争斗、仙人遗府、空间裂缝、心魔幻境……含**保底机制**，连黑必有奇遇。
- **战斗**：文字回合制，攻击 / 防御 / 法宝技能 / 丹药 / 逃跑，伤害含五行克制与随机浮动。
- **坊市**：货架随机刷新、价格浮动；魔劫入侵时物价飙升。可买丹药、材料、法宝、功法、配方，半价回收。
- **炼丹 / 炼器**：按丹方器图投入材料，神识与难度决定成率，产出有**品质波动**（低阶 / 正常 / 极品双份）。
- **背包**：分类物品、法宝装备与耐久、储物袋扩容；功法 / 称号提供被动加成。
- **成就与称号**：空血反杀、十连突破、炼丹宗师、白日飞升等隐藏成就。
- **混沌事件**：极低概率触发世界级「灵潮爆发」（修炼翻倍）、「魔劫入侵」（物价紊乱）。
- **每日活力**：主动修炼 / 探索 / 炼制消耗**活力**（行动力），跨自然日自动回满；活力耗尽时仅能挂机被动修炼、逛坊市等「部分事情」，符合修仙挂机节奏。
- **多存档**：支持 **5 个存档槽**，可新建 / 进入 / 删除 / 重开；旧版单存档首启自动迁移到槽 1。仍保留导出/导入字符串与**离线修炼收益**（上限 8 小时）。

## 项目结构

```
src/
  config.js          境界/灵根/数值公式（纯函数）
  data/
    items.js         丹药/材料/法宝/功法/杂物图鉴
    events.js        探索事件池（场景×境界×权重）
    enemies.js       妖兽动态生成
    recipes.js       丹方与器图
    characters.js    天赋/出身/形象（头像）/气运档次
  core/
    rng.js           可注入随机源（种子化/测试）
    player.js        玩家状态、衍生属性、背包、装备、创角、活力
    cultivate.js     修炼（被动/主动/顿悟）
    explore.js       探索抽取 + 奖励结算 + 混沌 buff
    battle.js        回合制战斗
    breakthrough.js  突破 + 心魔/天劫考验
    alchemy.js       炼丹/炼器
    market.js        坊市
    achievements.js  成就与称号
    save.js          多槽存档/导入导出/离线结算
  ui/
    dom.js           轻量 h() DOM 辅助
    style.css        竖屏移动端样式
    app.js           UI 控制器（存档选择/创角/状态栏/五页/弹窗/循环）
  main.js            入口：createGame(parent) 工厂
scripts/logic-test.mjs   纯逻辑自测（126+ 断言）
scripts/smoke-dom.mjs    jsdom 冒烟测试（需 npm i jsdom，覆盖存档/创角/活力 UI 流程）
```

## 设计原则

1. **碎片时间友好**：单次操作数秒，修炼可挂机。
2. **正反馈强烈**：突破、爆宝、奇遇有醒目动效与文案。
3. **风险与机遇并存**：探索可能遇险，也可能一夜暴富。
4. **足够随机但不致死**：保底机制、复活丹、重修 buff 给予挽回途径。
5. **修仙味要正**：境界名、事件文案模仿原作语境。
